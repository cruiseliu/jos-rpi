Detailed Description

The kernel shell.

The monitor use stdio to communicate with user, and is not interested in the implementation details of stdio. Thus stdio should be set up before starting the shell.

Function Documentation

int Monitor::backtrace ( int  argc,
char *  argv[] 
)

Display the stack backtrace.

Documentations are not always reliable. The official website[1] said stack frame layout below fp should be (pc,lr,sp,fp). However I disassambled my kernel and found the first instruction of each function is push {fp, lr}, which means fp[0] is lr and fp[-1] is old fp.

The arguments are not on the stack thus invisible to backtrace.

[1] http://infocenter.arm.com/help/index.jsp?topic=/com.arm.doc.dui0041c/Cegbajdj.html
